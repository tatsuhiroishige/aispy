## 2. 直近の session で完了したこと

### 2.1 MCP protocol 整理

- `htmlToText` を `htmlToMarkdown` (AI 用) + `htmlToTerminal` (TUI 用) に分離
- Fetch cache: `{ content, aiContent, imagePrologue, tokens }` で 両用途を separate
- Token-aware AI truncation (CJK 1.5 tokens/char 推定、20k tokens cap) で Claude Code の 25k cap を回避
- `fetch-update` IPC event で AI fetch でも TUI 段階更新

### 2.2 Section-based chunked fetch

- `core/sections.ts`: heading-based markdown splitter + TOC 生成
- `fetch` tool に `section` / `list_sections` / `offset` / `limit` param
- AI が大サイトを chunked retrieval できる

### 2.3 Bookmarks + Ctrl-R history search

- `~/.aispy/bookmarks.json` persistence (`core/bookmarks.ts`)
- Fuzzy search (`browser/historySearch.ts`): exact > starts-with > substring > fuzzy scoring
- `B` で bookmark toggle、`Ctrl-R` で overlay、↑↓/Enter で navigate

### 2.4 Form input MVP

- `browser/forms.ts`: extractForms / encodeFormData / buildSubmitUrl
- `navigator.submitForm` で GET query / POST urlencoded
- `components/FormModal.tsx`: field list、Tab で移動、Enter で submit
- `F` で page 上の form を編集

### 2.5 Image streaming (Option 2)

- `<img width/height>` hint で layout 安定化 (phase 切替で shift 減)
- `renderHtmlToTerminalStream`: text → partial × N → final、4 枚ずつ decode & yield
- `Tab.decodeProgress` で `UrlBar` に "◌ images 4/19" 表示

### 2.6 SPA 検知の厳格化

- Content 長 3 bucket 判定 (<200 / 200-1500 / ≥1500)
- 弱い marker (`window.__`, `createElement`) 削除、強い marker (`__NEXT_DATA__`, 空 mount div) のみ残す
- Jina fallback 使用時も TUI には **常に terminal stream を流す** (誤検知時の TUI 被害を断つ)

## 5. Roadmap (impact 順)

### Step 1: Floats (次 session)

`src/core/floats.ts` を新規 + `bfc.ts` / `ifc.ts` に統合:
- `ExclusionList`: 既存 float の矩形を保持
- `findNextFloatOffset(y, width)`: inline flow 中に空き矩形を探す
- `clear: both`: exclusion list を一旦無視して y を送る
- `shrink-to-fit` width 計算 (float の intrinsic width)

完了時: Wikipedia の infobox が右寄せ、news hero image が float:left に配置される。

### Step 2: Borders + background

`cssResolver.ts`:
- `border-style/width/color`, `background-color` を `ComputedStyle` に追加

`paint.ts`:
- block box の state から矩形取得 → bg cell を fill
- border-style を box-drawing chars に変換 (`┌─┐│└┘`)
- 角の結合 (`┼` `┤` 等 neighbor 見る) は後回し、単純 4-line + 4-corner から

完了時: 注意 box、button、table 枠、card 背景が見える。

### Step 3: Form visual

`cssResolver.ts` の default style に input/textarea/button 追加 (border + padding)。`paint.ts` は Step 2 で自動対応。Enter で既存 `FormModal` を open。

### Step 4: CSS grid

Yoga 使うか、自前 track-sizing algorithm を書くか判断。Amazon / docs の多列 layout 対応。

### Step 5: Link focus highlight

`App.tsx` で現在選んでる link URL を保持 → `paint.ts` で該当 atom の Format を red に override。

### Step 6: Margin tuning

`cssResolver.ts` の default margin を Chawan 相当に詰める。
